Package: io.github.longfish801.tpac.tea

[Groovy] Trait TeaServer

    • io.github.longfish801.tpac.tea.TeaServer

    • tpac文書を保持する特性です。
      複数の tpac文書を保持できます。
      スカラー値として指定したハンドルのパス文字列で 他の tpac文書を参照するときには、 参照元および参照先が同じサーバで保持されている 必要があります。
      独自のDSL利用のため、解析器（TeaParty）を 変更したい場合は newParty()を、 宣言のタグに応じて生成器（TeaMaker）を 変更したい場合は newMaker(String)をオーバーライドしてください。

      Version:
      0.3.00 2020/05/23
      Authors:
      io.github.longfish801

        • TeaMaker newMaker(String tag)

          宣言のタグに対応する生成器を返します。
          現状、宣言のタグとは無関係に TpacMakerインスタンスを返します。
          宣言のタグに応じて生成器を変更したい場合は本メソッドを オーバーライドしてください。
          ひとつの tpac文書毎に本メソッドで新たなインスタンスを ひとつ生成してください。

          Parameters:
          tag - 宣言のタグ
          Returns:
          TeaMaker

        • TeaParty newParty()

          tpac文書の解析に用いる解析器を返します。
          現状は TpacPartyインスタンスを返します。
          独自のDSL利用のため、解析器を変更したい場合は 本メソッドをオーバーライドしてください。
          ひとつの解析対象毎（soak(def)メソッド実行毎）に 本メソッドで新たなインスタンスをひとつ生成してください。

          Returns:
          tpac文書の解析器

        • TeaHandle solvePath(String path)

          tpac文書からパスに対応するハンドルを参照します。
          パス区切り文字で分割した先頭の識別キーと、残りのパスに分割します。
          残りのパスがなければ、先頭の識別キーと一致する宣言を返します。
          残りのパスがあれば、先頭の識別キーと一致する宣言にそれを新たなパスとして依頼します。

          Parameters:
          path - パス
          Returns:
          パスに対応するハンドル（該当するハンドルがなければnull）
          Throws:
          TpacHandlingException 統語的にありえないパスです
